UX Vision

Fractal Glass embraces a progressive disclosure design philosophy, presenting simple, immediate actions to new users while revealing powerful customization options as users grow more confident. The interface balances visual appeal with functional clarity, using the glass effects themselves as inspiration for the UI design language.

ðŸŽ¯ Core Design Principle

"One click for magic, infinite clicks for mastery"

New users should achieve professional results in seconds. Power users should never feel constrained by the interface. The UI adapts to user expertise through progressive disclosure and intelligent defaults.

Design Principles

1. Immediate Value

Users see results within 2 seconds of opening the plugin. The preset gallery is the hero element, showcasing visual examples that speak louder than descriptions.

2. Visual Feedback First

Every parameter adjustment shows its effect immediately in the preview. Users learn by seeing, not by reading documentation.

3. Respectful Complexity

Advanced controls are hidden by default but never removed. Power users find depth; casual users find simplicity.

4. Forgiving Interactions

Every action is reversible. Experiments are encouraged through non-destructive editing and unlimited undo.

5. Aesthetic Inspiration

The plugin interface itself uses subtle glass-like effects, creating cohesion between tool and output.

Interaction Patterns

Preset Selection

Behavior: Single-click selects preset (highlight border). Selection persists until another preset chosen or "Clear" clicked.

Visual Feedback:

  • Default: 2px solid #e2e8f0 border
  • Hover: 2px solid #cbd5e0 border + subtle scale (1.02)
  • Selected: 4px solid #667eea border + drop shadow

Accessibility: Focus state identical to selection state, keyboard navigation with arrow keys

Slider Controls

Behavior: Click-drag or click on track to jump. Arrow keys for fine adjustment (Â±1 unit).

Visual Feedback:

  • Track: 4px height, rounded corners, #e2e8f0 background
  • Fill: Progressive color from #667eea to #764ba2
  • Thumb: 16px circle, white with shadow, scales to 18px on hover

Live Updates: Preview updates debounced at 300ms during drag

Accessibility

WCAG 2.1 AA
Keyboard Navigation
Screen Reader Optimized

Color Contrast

All text meets WCAG 2.1 AA standards:

  • Normal text: Minimum 4.5:1 contrast ratio
  • Large text (18px+ or 14px+ bold): Minimum 3:1 contrast ratio
  • UI components: Minimum 3:1 contrast ratio

Keyboard Navigation

Key Action Context
Tab Move focus forward All interactive elements
Shift + Tab Move focus backward All interactive elements
Enter / Space Activate element Buttons, presets, toggles
Arrow Keys Navigate within component Preset grid, dropdown menus, sliders
Escape Close / Cancel Modals, dropdowns, popups
Cmd/Ctrl + Z Undo last action After applying effect

Screen Reader Support

  • ARIA Labels: All interactive elements have descriptive labels
  • Role Attributes: Proper semantic HTML with ARIA roles where needed
  • Live Regions: Status updates announced (e.g., "Effect applied successfully")
  • Focus Management: Focus trapped in modals, restored on close
  • Image Alt Text: Preset thumbnails have descriptive alt text

Focus Indicators

All focusable elements have clear focus indicators:

  • Primary: 2px solid outline in primary color with 2px offset
  • High Contrast Mode: Increased to 3px solid outline
  • Never Removed: outline: none is forbidden

Voice and Tone

  • Friendly but Professional: Approachable without being overly casual
  • Concise: Every word earns its place
  • Action-Oriented: Verbs over nouns, active voice
  • Empowering: "You can..." rather than "It's possible to..."

Tooltips

Tooltips appear on 500ms hover delay, dismissed on mouseout or 5 seconds:

  • Stripe Count: "Number of vertical stripes. More stripes = smoother effect but slower generation"
  • Blur Radius: "Glass diffusion amount. Higher values create softer, more diffused effects"
  • Color Space: "LAB creates most natural color transitions. HSL maintains saturation better than RGB"
  • Blend Mode: "How the effect mixes with layers below. Try Overlay for subtle integration"

Animation and Transitions

Animation Principles

  • Purpose-Driven: Every animation serves a function (feedback, guide attention, show relationships)
  • Subtle: Never distracting or unnecessary
  • Fast: Most animations 150-250ms; nothing over 400ms except large transforms
  • Natural: Ease-out for entrances, ease-in for exits

Loading States

  • Spinner: Rotating gradient ring, 300ms per rotation
  • Progress Bar: For operations over 5 seconds, shows estimated completion
  • Skeleton Screens: Preset gallery loads with placeholder cards (pulsing animation)
Reduced Motion: Respect prefers-reduced-motion media query. Disable all non-essential animations, reduce durations to 50ms max.

Performance Considerations

UI Performance Targets

  • Initial Load: Plugin UI visible within 500ms
  • Preset Gallery Render: All 8 presets displayed within 200ms
  • Parameter Adjustment: UI responds within 16ms (60fps)
  • Preview Update: Debounced to 300ms during continuous adjustment
  • Mode Switch: Quick Apply â†” Advanced transition in under 200ms

Optimization Strategies

  • Lazy Loading: Advanced controls only loaded when Advanced mode activated
  • Virtual Scrolling: For preset libraries exceeding 50 items
  • Image Optimization: Preset thumbnails cached after first generation
  • Debouncing: Slider inputs, color pickers, and text inputs debounced 300ms
  • Progressive Enhancement: Core Quick Apply functionality works even if Advanced mode fails to load
